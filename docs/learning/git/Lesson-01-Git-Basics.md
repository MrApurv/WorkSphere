## Date

## Objective

Understand why Git is used and how version control works.

---

## What is Git?

Git is a distributed version control system used to track changes in source code.

It helps developers:

- Track history
- Collaborate with teams
- Restore previous versions
- Work safely using branches

---

## Why Git?

Without Git

Developer edits file

↓

Mistake

↓

No previous version

With Git

Developer edits file

↓

Commit

↓

Restore anytime

---

## Important Git Commands

git status

Checks current project status.

---

git add .

Stages all modified files.

---

git commit -m "message"

Creates a snapshot of current changes.

---

git push

Uploads commits to GitHub.

---

git pull

Downloads latest commits from GitHub.

---

git branch

Shows available branches.

---

git switch branch-name

Switches branches.

---

git switch -c feature/login

Creates and switches to a new branch.

---

git log --oneline

Displays commit history.

---

## Git Workflow

Modify Code

↓

git status

↓

## Key Learning

Git tracks project history.
